Challenge

Design a tutorial for an existing live tool

Data Driven Detroit presented our team with the 'simple' challenge: design a user friendly learning tutorial for an open source database tool to be used by non-profit organizations, researchers, teachers, policymakers, and residents.

Why?

"This is a free tool. We want people to use to learn more about their community from these comprehensive databases. But from our workshops it seems like the learning curve is too much for our diverse audience."

The complexities of an open source tool and vast of amount of data available was impacting user retention and interest. The existing tutorial doesn't seem to help.



Existing Tutorial

What do users have today?

Right now the tutorial is a scrolling one page website with text boxes introducing new features.

The tool and tutorial are very similar visually and both rely heavily on scrolling (which also translates to user interest in exploring).

Methodology

Identifying key interaction point in the tool and tutorial

As we started to explore the tool we realized that there was already an existing tutorial embedded on the tool. Hence we started with an analysis of interaction points and break downs in the existing tool as well as the tutorial. Next, we analyzed existing user research reports and conducted stakeholder interviews.

This helped us understand the tool and discover existing breakdowns.

To analyze this qualitative data we created a affinity wall and categorized our annotated notes.

Insights

Narrowing down the scope and key issues to target

The main constraint we had to work around was not suggesting changes to the tool but creating a tutorial with the existing tool.

Though we identified multiple usability issues with the tool the project constraint of not being able to change the tool meant we had to use the tutorial to tackle some of those major issues.
