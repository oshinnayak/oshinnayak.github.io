Creating User Archetypes
GOAL: Create a UX Research plan and define user archetypes needed.

Once I took upon the task of analysing interviews transcripts and used Nvivo to open code them, patterns started arising. We hoped these insights would shed light on what users like and what they do not understand. Suprisingly,my analysis led to one an unexpected insight.
The interview script needs to be modified. The project’s end result was changing the project itself.
In the infrasture world, role or position of users and their work aren’t mapped consistently through the industry. This led us to also consider modifying the interview scripts and the study plan itself. Instead of just interviewing customers we decided to talk to employees who had been in the firm for years and could shed light on issues customers faced in detail.
Redesigning File Servers
GOAL: To visually communicate the File server creation process in Nutanix nodes so that users can efficiently utilize available resources both in terms of cost and storage. Overall this would help increase user knowledge of the process itself.

The most challenging part of my internship was to create high fidelity prototypes for Nutanix Prism components. I had to redesign a perfectly functional and usable component.

How did I design a new flow concentrating on visual communication?

  • Analysed the usability data from .NEXT Nice and DC conferences for previous Visual flows for AFS which had not gone live.
  • Used maze.design to share the redesigned prototypes and collect comprehensive feedback to use for future iterations. (See screenshots below.)
  • Conducted expert usability tests with UI and backend engineers of File Servers.
Visualize your Network
GOAL: In a three day Hackathon my team and I tried to answer a common yet complex question of the Networks world. Which node is not working?

From datacenter admins we learned that error dectection is a time consuming task that could range from hours to weeks. Root cause analysis is as scary as the physical datacenters.

Mapping physical connections in a datacenter to a visual network topology would help reduce some of this complexity. Our proposed design used simple shapes (nodes) and red colored dots (errored link source) to indicate overall state of the datacenter. Our deign rationale was based on the following

To monitor a complex system where mutiple things can go wrong, an efficient error dectection system could be static when all systems are up. In case of an error, it could indicate it through motion, color, and concentrated activity in the errored region.
Reduce the cognitive load by displaying details in progressive manner.

Due to the NDA agreement the designs cannot be shared here.
